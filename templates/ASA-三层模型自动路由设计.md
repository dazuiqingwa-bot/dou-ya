# 1. 目标（Outcome）
在不需要老板手动下指令切换模型的前提下，ASA 能根据任务类型自动选择最合适模型，实现“质量 × 成本 × 速度”平衡。

# 2. 三层模型策略（V1）

## 第一层：决策搭档（高价值、低频）
- 默认模型：**Claude**
- 适用任务：战略判断、多视角分析、方向决策、关键取舍
- 触发信号（示例）：
  - “要不要做 / 先做哪个 / 路线怎么选”
  - 涉及高不确定性与长期影响

## 第二层：问题解决者（中高频）
- 默认模型：**ChatGPT**
- 适用任务：技术排错、流程拆解、方案比较、可执行步骤
- 触发信号（示例）：
  - “怎么做 / 分步骤 / 帮我排查”
  - 需要稳定输出和较快迭代

## 第三层：执行者（高频、低复杂）
- 默认模型：**本地模型（Mac mini 部署后）**
- 适用任务：重复整理、格式转换、草稿整理、批量执行
- 触发信号（示例）：
  - 明确规则、低风险、可模板化

# 3. 自动路由规则（先简单后复杂）
1. **先判层级，再选模型**（禁止“先选模型后找理由”）
2. 置信度不足时，默认上浮一层（第三层→第二层，第二层→第一层）
3. 涉及重大决策/高风险内容，强制走第一层
4. 执行阶段若连续失败 ≥2 次，自动升级到第二层处理
5. 如果第一层超预算或拥堵，临时回退到第二层并标记“待复盘”

# 4. 成本与性能优化（Mac mini 后重点）
- 目标：高频任务尽量本地化，关键任务保留云端高质量模型
- 建议指标：
  - 路由命中率（是否分到正确层级）
  - 单任务平均成本
  - 任务完成时长
  - 失败升级率（第三层→第二层 / 第二层→第一层）

# 5. 存储策略（Reminders vs Obsidian）
- **Reminders**：只放“有截止时间、必须触发执行”的提醒
- **Obsidian**：放策略文档、规则、复盘、版本变更
- 本项目落地方式：
  - Reminder：`ASA 三层模型自动辨别与切换方案（Obsidian初稿）`（Due: 2026-04-30）
  - Obsidian：本页面作为主文档，后续迭代保留版本记录

# 8. 风险清单
- 风险1：自动判断误分层 → 缓解：设置信心阈值和升级机制
- 风险2：本地模型质量不稳定 → 缓解：限定第三层任务边界
- 风险3：规则过度复杂 → 缓解：先做 V1 极简规则，再迭代
