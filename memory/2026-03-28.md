## 下午：飞书语音转写（STT）修复完成

### 根本原因
- OpenClaw 的 media understanding provider registry 有 bug：OpenAI provider 通过 plugin 注册，但 `buildMediaUnderstandingRegistry` 里的 `PROVIDERS` 默认数组只有 groq 和 deepgram
- 手动配置 `models: [{provider: "openai"}]` 时，provider 在 registry 里找不到，被静默跳过
- auto-detection 模式理论上会走 `resolveKeyEntry` 检测 openai key，但实际测试中同样不工作（原因未完全确认）

### 最终修复方案
- 用 CLI 模式绕过 plugin registry 问题
- 写了 shell 脚本 `~/.openclaw/workspace/scripts/transcribe.sh`，用 curl 直接调 OpenAI transcription API
- 配置：`tools.media.audio.models = [{type: "cli", command: "...transcribe.sh", args: ["{{MediaPath}}"]}]`
- 验证：老板 18:03 发语音，豆芽正确转写并回答 ✅

### echoTranscript 状态
- `echoTranscript: true` 配置中，但豆芽没有先发 📝 确认消息
- 可能是 feishu channel 不在 `isDeliverableMessageChannel` 检查里，或 echo 路径有别的问题
- 不影响转写主流程，暂不追踪
