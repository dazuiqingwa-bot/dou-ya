# skill: routing
# 何时加载：需要判断是否路由给 Learning Center 时

## 优先级

图片触发 > 作答触发 > 文字描述触发

消息里有图片，直接走图片流程，不走其他判断。

---

## 触发 1：图片（最高优先级）

收到图片时，第一个动作必须是调用 `sessions_send`，把题目信息发给 Learning Center：
- session key：`agent:learning-center:main`
- 内容：学科、题目描述、知识点、图片识别内容

调用后不等待返回，立刻开始文字讲解。

**禁止在调用 sessions_send 之前输出任何分析或答案。**

入库完成后，只做一句轻量确认：
「这题我先帮你记下了，我们现在看第一步。」

然后进入五次发送讲题流程（见 teaching.md）。

拍照 = 有错题 = 自动入库，不需要禾禾手动说"记进去"。
语音 + 图片同时发来时，语音是补充说明，图片是主体。

---

## 触发 2：作答识别

同时满足以下两个条件时，路由给 Learning Center：

条件 A（有作答内容）：
- 包含计算过程、数字、公式
- 或明确说"我的答案是""我算的是""我觉得是"
- 或语音说了类似内容

条件 B（有上下文关联）：
- 最近豆芽刚发过练习题
- 或禾禾提到"这道题""刚才那道""第几题"

触发后执行流程：
1. 提取禾禾的作答内容
2. 判断对错，分析错因
3. 用 exec 调用脚本更新 Notion 掌握状态（不可跳过）：
   - 脚本：`python3 ~/.openclaw/workspace/scripts/notion-update-status.py <page_id> <新状态> <下次复习日YYYY-MM-DD>`
   - 做对（原"未掌握"）→ "半掌握"，复习日 = 今天 + 3天
   - 做对（原"半掌握"）→ "已掌握"，复习日 = 今天 + 7天
   - 做错 → 保持"未掌握"，复习日 = 明天
4. 用学姐口吻反馈批改结果（见 teaching.md 批改反馈模板）

---

## 触发 3：文字描述错题

同时满足：
- 提到具体学科 + 错了/不会/不理解/扣分
- 有题目描述/知识点/她的答案

触发后：信息不完整才追问，否则直接路由给 Learning Center 做错因分析 + 变式题生成 + 写入 Notion。

---

## 不触发的情况

- "数学今天好难啊" → 没有具体错题，正常聊天
- "帮我讲一下这个概念" → 直接讲解，不路由
- 纯聊天、创作、探索类话题 → 探索模式，不路由

---

## 总流程顺序（不允许跳步）

1. 收到题目 / 错题 / 图片
2. 判断是新题、旧题、还是题库回看
3. 错题或需要沉淀 → 先入 Learning Center / Notion
4. 再用五次发送给禾禾讲题
5. 讲完后给练习题（不是一上来就塞）
6. 禾禾作答后批改，更新掌握状态
7. 复习旧题时，先定位原记录，再继续讲和出题

---

## 查原题的话术规则

定位到记录时：
「我把你上次那题找到了，我们接着这个继续。」

没定位到时：
「我现在还没对上是哪一题，你给我一个线索，我马上继续找。」

禁止说"没有权限""Notion 不让我看""系统查不到"，除非后台确实返回了权限错误。
